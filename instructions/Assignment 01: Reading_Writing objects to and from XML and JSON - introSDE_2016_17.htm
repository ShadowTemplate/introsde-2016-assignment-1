Assignment 01: Reading/Writing objects to and from XML and JSON

Introduction to Service Design and Engineering | University of Trento | Webpage

  • Lab Session 02: Webpage | Source Code

  • Lab Session 04: Webpage | Source Code

Technologies:

  • Java
  • XML/XSD
  • JSON
  • XPATH
  • JAXB/Jackson

Guiding Notes

  • Using the Health Profile Reader/Writer examples we have used in the lab, replace the array list or hash map we used as database with a xml file as follows

    <people>
    <person id="0001">
        <firstname>George R. R.</firstname>
        <lastname>Martin</lastname>
        <birthdate>1984-09-20T18:00:00.000+02:00</birthdate>
        <healthprofile>
            <lastupdate>2014-09-20T18:00:00.000+02:00</lastupdate>
            <weight>90</weight>
            <height>1.70</height>
            <bmi>31.14</bmi>
        </healthprofile>
    </person>
    <!-- add more people to the db --> 
</people>

  • Extend the example above to include 20 people (maybe your friends with fake names)

Based on Lab 3
  1. Use xpath to implement methods like getWeight and getHeight (

    getWeight(personID) returns weight of a given person, the same for getHeight

    )
  2. Make a function that prints all people in the list with detail
  3. A function that accepts id as parameter and prints the HealthProfile of the person with that id
  4. A function which accepts a weight and an operator (=, > , <) as parameters and prints people that fulfill that condition (i.e., >80Kg, =75Kg, etc.).
Based on Lab 4
  1. Create the XML schema XSD file for the example XML document of people.
  2. Write a java application that does the marshalling and un-marshalling using classes generated with JAXB XJC.
  3. Make your java application to convert also JSON
Evaluation script
Create a target in your build.xml file execute.evaluation (make sure it does all necessary compilation/generation before), which:
    1. runs instruction 2 based on Lab 3
    2. runs instruction 3 based on Lab 3 with id = 5 (make sure you have a person with such an id)
    3. runs instruction 4 based on Lab 3 with weight > 90
    4. runs instruction 2 based on Lab 4 (marshaling to XML - create 3 persons using java and marshal them to XML) - please print the content and save to .xml file
    5. runs instruction 2 based on Lab 4 (un-marshaling from XML)
    6. runs instruction 3 based on Lab 4 (marshaling to JSON - create 3 persons using java and marshal them to JSON) - please print the content and save to .json file
Delivery (due on 25-Oct 11.59PM)
  1. Make sure you can run your code (ant execute.evaluation) on your machine
  2. Create a repository in Github, under your account (call it introsde-2016-assignment-1)
  3. Push your code (no /build, no libraries, no .gitignore, no IDE files (e.g. .idea, .project etc)) to this repository 
  4. Create a readme.md file in Github and add there some description of the project (1 paragraph about the code, 1 paragraph about the tasks the code does, and 1 paragraph - how to run it)
  5. Make sure you can do git clone {your repository}, do ant execute.evaluation and everything works well

Assignment Evaluation

  • The assignment will be evaluated in terms of:
    • Documentation: do you provide a proper documentation to understand your work? Try to leave comments in the code, indicating what certain parts of the code do.
    • Completeness: did you deliver all that was asked?
    • Functioning: does it work? Does it do what it was requested?
    • Organization: the easier you make it for us to understand what you did and how to run it, the more chances for you to have a better mark (i.e., there is self-explained ant build file to automate things, files are structured in orderly manner, source code is readable, dependencies are managed with ivy or other dependency manager, etc.)
